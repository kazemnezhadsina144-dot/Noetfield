Deliverables (D0–D5)

Named, exportable outputs — designed to remove ambiguity, reduce risk, and make Copilot decisions defensible.

D0
Executive summary (1 page)
RAG posture + decision requests + what changes now vs later.
D1
Surface inventory & enablement map
Which Copilot surfaces are enabled, for whom, and under what scope.
D2
Oversharing heatmap (Top 10)
Exposure paths driven by current permissions posture (Teams/SharePoint/OneDrive).
D3
Data boundaries pack
Sensitive domains + boundary rules + “red-line” prompt/content guidance v1.
D4
Evidence readiness sheet
Signals, retention posture, export steps, and owners — what counts as proof.
D5
Connector / extension controls register
Allowed vs restricted, change control, exceptions, and break-glass rules (if applicable).
30/60/90 plan

A ranked action plan with owners and sequencing for safe adoption.

  • 30: quick wins + top-risk exposure controls.
  • 60: boundaries, policy alignment, connector control hardening.
  • 90: evidence maturity, cadence, and re-scan readiness.
5–7 day structure

Designed to be decisive: scope → posture → evidence → board-ready pack.

Day Focus Output
0–1 Scope lock + access readiness Engagement boundaries, target surfaces, access checklist, export plan
2–3 Surface + enablement mapping Inventory of enabled experiences, audience scope, ownership map
3–4 Exposure + boundary review Oversharing paths, sensitive domains, boundary posture snapshot
4–5 Evidence readiness capture Signals, retention, exportability, evidence steps + owners
5–7 Synthesis + delivery Executive summary + ranked actions + 30/60/90 plan
Access required

Read-only is preferred. No content sampling unless explicitly authorized.

  1. M365 admin contact (tenant posture / enablement).
  2. Security / compliance contact (labels/DLP, retention, audit posture).
  3. Business owner for Copilot adoption (decision boundaries and priorities).
Limitations

Assessment + evidence-readiness outputs. Implementation/remediation and adversarial testing are separate scopes.

Add-ons

Optional next steps after QuickScan, if needed.

  • Remediation sprint (2–3 weeks) + re-scan + updated evidence pack.
  • Quarterly evidence refresh (delta report) for ongoing oversight.
  • Copilot Studio governance program (if custom copilots/agents are in scope).
  • Connector/vendor egress review for high-risk integrations.
