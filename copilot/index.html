2) What you get

Forwardable artifacts (not slides)

Outputs are packaged for internal routing: exec review, security review, legal review, and procurement filing.

Readiness outputs (2/4/6 weeks)
Decision-grade baseline with a clear owner map and evidence posture.
  • Board memo: decision points, risk statement, recommended actions.
  • Control posture: what is on/off, allowed/blocked, reviewed/monitored.
  • Exposure surfaces map: boundary conditions and priority hotspots.
  • Evidence index: what exists vs missing, mapped to procurement confidence.
  • 30/60/90 plan: sequencing, owners, and go/no-go gates.
QuickScan outputs (5–7 days)
Rapid signal that reduces uncertainty and aligns owners before escalation.
  • Scope lock: explicit boundary + assumptions.
  • Oversharing heatmap: risk themes and high-risk surfaces.
  • Connector/extension notes: allow/block/review-first posture.
  • Evidence checklist: procurement-ready list of what to file today vs gaps.
  • Decision options: proceed / pause / remediate, with next-step logic.

3) Process

60-second runbook (buyer-visible)

Designed to reduce friction between IT, Security, Legal, and Procurement.

  • Non-confidential intake: intent, stakeholder map, access posture (no secrets).
  • Scope lock: explicit boundary (who/where/what) + assumptions + decision constraints.
  • Evidence requests: minimal, prioritized collection (procurement-safe checklist).
  • Checkpoint: align owners + must-not-expose surfaces + connector/extension posture.
  • Decision pack: findings + risk hotspots + 30/60/90 plan + proceed/pause/remediate options.
Measured on outcomes: confirmed risk-reduction actions + decision readiness, not document volume.

6) Boundaries

Non-confidential first contact

Public forms are intentionally safe for early-stage routing.

Allowed in first message
High-level context that helps scope and route the engagement.
  • Tenant size (approx.), business unit count, and rollout intent
  • Stakeholder roles (IT/Security/Legal/Procurement)
  • Target timeline and known constraints
  • General data sensitivity categories (no identifiers)
Not allowed in first message
Sensitive content must wait for a secure channel.
  • Customer data, employee data, personal identifiers
  • Secrets, credentials, tokens, internal incident details
  • System screenshots with identifying information
  • Any regulated datasets unless explicitly arranged
Where to start: Gate (non-confidential) → scope lock → secure channel (if fit) → evidence request (minimal, prioritized).
Reminder: Not legal advice.

8) FAQ

Common buyer questions

Is the first intake confidential?
No. Public intake is non-confidential by design. If there is a fit, a secure channel and access plan follows.
Do we need deep access on day one?
Not necessarily. The motion starts with scope lock and an access readiness plan, then escalates access only as required.

Readiness and QuickScan are decision + evidence first. Remediation/implementation can follow as a scoped sprint if needed.
Can procurement use the outputs?
Yes. Outputs are designed to be forwarded, filed, and referenced in approval workflows.
Which should we choose?
