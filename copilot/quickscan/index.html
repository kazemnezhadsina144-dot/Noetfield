Copilot QuickScan.

Board-ready evidence for Copilot adoption — delivered in 5–7 business days. QuickScan produces an evidence-first governance pack covering Copilot surfaces, oversharing exposure paths, sensitive-data boundaries, audit/retention posture, and extension/connector controls.

Start intake  ·  Copilot hub  ·  Baseline map

What is scanned

Fixed scope across five pillars.

1) Copilot surface inventory + enablement map

  • Which Copilot experiences are active (M365 Copilot surfaces; Copilot Studio/custom copilots if present).
  • Who is enabled, and under what licensing/tenant scope.

2) Permission sprawl + oversharing exposure paths

  • SharePoint / OneDrive / Teams sharing model review: high-risk sites, anonymous/external links, broad groups, inherited permissions.
  • Where Copilot could amplify unintended access based on existing permissions.

3) Data boundaries + sensitive content controls

  • Data classification alignment for high-risk domains (HR / finance / legal / client).
  • DLP and sensitivity labels posture (governance-level) and “red-line” rules for prompt/content handling.

4) Logging, retention, and evidence readiness

  • Which usage/audit signals exist, retention posture, and exportability.
  • Minimum evidence set to answer audit / board / client due diligence questions.

5) Extension/connector/custom Copilot risk controls

  • Plugins/connectors/custom copilots (if present): allowed vs restricted tool access.
  • Change control for copilots/prompts and exception / break-glass rules for high-risk actions.

What the client receives

  • D0 — Executive Summary (1 page, RAG): overall status + 5-pillar RAG + decision requests.
  • D1 — Copilot Surface Inventory & Enablement Map.
  • D2 — Oversharing Heatmap (Top 10) + ranked actions (quick wins vs projects).
  • D3 — Data Boundaries Pack + Red-Line Rules v1 (publish-ready internal memo).
  • D4 — Evidence Readiness Sheet: signals, retention posture, export steps, owners.
  • D5 — Extension/Connector Control Register: allowed/restricted tools + change control + exception protocol.
  • 30/60/90 plan: quick wins + governance milestones and owners.

Process and timing

  • Day 0–1: scope lock + access readiness.
  • Day 2–4: inventory + posture review + evidence capture.
  • Day 5–7: synthesis + delivery of board-ready pack.

Access required

Read-only administrative access is preferred where possible, sufficient to export configuration and reporting signals across in-scope Microsoft 365 workloads. No content sampling unless explicitly authorized.

  • M365 Admin
  • Security / Compliance lead
  • Business owner (Copilot adoption owner)

Limitations

QuickScan provides assessment and evidence readiness outputs. Remediation implementation, content-level review, and adversarial testing are available only as separate scopes.

Add-ons

  • Remediation Sprint (2–3 weeks) + re-scan
  • Copilot Studio deep governance program
  • Quarterly evidence refresh (delta report)
  • Connector/vendor egress review
