What this lane is

A standardized portfolio motion that helps cohort companies clear enterprise procurement and governance friction for AI/Copilot decisions. Designed for repeatability across cohorts without oversharing.

  • Program-level enablement: consistent artefacts and routing discipline.
  • Non-confidential boundary: no credentials, tenant identifiers, customer content, or regulated personal data in email or web forms.
  • Routing discipline: buyers use Gate; procurement uses Invoice/PO lane; program uses Programs lane.
Model A

Program contract (portfolio package)

Single engagement with the program for a defined cohort window and standardized outputs.

  • Best when: program sponsors and wants consistent artefacts
  • Procurement: program procurement
  • Handoff: per-company RID created at nomination
Model B

Company-by-company (buyer contracts)

Each portfolio company runs its own Gate → Procurement flow using the program’s routing rules.

  • Best when: companies have independent budgets/procurement
  • Procurement: each company procurement
  • Program: deal registration + governance readiness standard

Default cohort deliverables (board-safe)

  • Per-company snapshot: QuickScan-style pack (non-confidential intake + evidence boundary map)
  • Decision artefact format: memo structure + MAP table conventions + evidence ID starter
  • Control baseline: lightweight taxonomy aligned to procurement language
  • Program summary (optional): anonymized pattern report (non-confidential)

Boundary rule: no credentials, tenant identifiers, customer content, or regulated personal data in public pages or emails.
