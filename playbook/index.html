The pattern

The goal is not to write perfect policies. The goal is to make AI use governable — with clear scope, owners, controls, and proof.

Phase 1: Baseline
0–30 days
  • Material AI inventory (what runs in prod, what influences decisions).
  • Owner map: accountable people, escalation, approval boundaries.
  • Minimum control baseline (access, logs, data handling, retention).
  • Decision framing: what boards must explicitly approve vs delegate.
Phase 2: Evidence
31–60 days
  • Evidence map: what counts as proof and where it lives.
  • Standard artefact pack (configs, screenshots, exports, runbooks).
  • First Trust Ledger entries for highest-impact use cases.
  • Internal “board memo” draft with options and trade-offs.
Phase 3: Cadence
61–90 days
  • Governance cadence (monthly ledger update, quarterly board review).
  • Change control (prevent drift; document exceptions).
  • Partner lanes for implementation (integration, delivery, channel).
  • Incident drill: prove you can reconstruct decisions after the fact.
Output: Board-ready package
deliverables
  • Control baseline with owners and evidence expectations.
  • Starter Trust Ledger entries (material use cases first).
  • Board memo: options, priorities, and next-step plan.
  • Implementation routing: who can ship controls + evidence trails.
Standard artefacts

Use these to keep everything auditable and readable (without bureaucracy).

Artefact Purpose Evidence expectation
AI inventory + materiality thresholds Know what matters and why List of systems + owners + decision impact notes
Control baseline Minimum viable governance Config screenshots/exports + log retention proof
Trust Ledger entry set Operationalize governance Use-case description + controls + proof references
Board memo Make decisions explicit Options, trade-offs, approval boundaries
Change control + exception log Prevent drift Who approved exceptions and when + follow-up plan
Open Gate Trust Brief Trust Ledger
Implementation lanes: if you need partners to wire controls and evidence trails, use Partners Gate and route via the right lane: Integration, Delivery, Channel.
